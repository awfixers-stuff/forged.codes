# About Forged Codes

Forged Codes provides backend infrastructure for version control, package registries, CI/CD, and security. We run the services that power code collaboration — you build the frontends and experiences your users see.

## What We Build

Our flagship product, **Grip**, is a version control system that combines Git compatibility, Mercurial's branching model, and Graphite's stack-based pull request workflows. It has every feature that GitHub and GitLab advertise — but no graphical interface.

The AWS model, applied to code: we provide the infrastructure, you provide the experience.

## Why No GUI?

Because building a great frontend for version control shouldn't require also building the backend. GitHub and GitLab are vertically integrated — they control both the UI and the infrastructure. That means you're locked into their design decisions, their feature cadence, and their pricing.

With Grip, you own the front door. Your brand, your design, your user experience. We handle the plumbing: hosting, version control operations, pipelines, registries, and security scanning.

## The Forged Codes Registry

The Registry is our package hosting platform — one of the services that runs on Grip infrastructure. It provides:

- **Secure package hosting** — npm, PyPI, Maven, Docker, and more
- **Fine-grained access controls** — Organization, team, and package-level permissions
- **Audit trails** — Every publish, install, and access decision is logged
- **CI/CD integration** — Packages flow through your pipelines automatically

## Our Approach

### Infrastructure First

We believe code infrastructure should be invisible. Developers shouldn't have to think about where their repos are hosted, how their CI pipelines scale, or whether their packages are secure. Grip handles all of that.

### API-Driven

Every feature in Grip is accessible through a well-documented API. If a human can do it through a GUI, a program can do it through our API — because the API came first.

### Open Where It Matters

Grip's protocols are Git-compatible. If you can push to GitHub, you can push to Grip. We don't believe in lock-in at the transport layer. The value is in the infrastructure and the operations, not in proprietary formats.
